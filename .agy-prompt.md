### Fix 2 (MEDIUM) — Unsafe Process Status Checks on Empty PID Variables

File: .agents/skills/start-backends/SKILL.md

In the `cleanup()` function (around lines 62-67), the loop checks `kill -0 "$LLM_PID"` / `kill -0 "$STT_PID"` / `kill -0 "$TTS_PID"` without guarding for empty PID variables. If a service failed to start early, these PIDs are empty strings, and `kill -0 ""` prints "illegal pid".

Fix: Guard each PID check so empty PIDs are treated as "already stopped". Replace:
```bash
  for _i in $(seq 10); do
    if ! kill -0 "$LLM_PID" 2>/dev/null && ! kill -0 "$STT_PID" 2>/dev/null && ! kill -0 "$TTS_PID" 2>/dev/null; then
      break
    fi
    sleep 1
  done
```
With:
```bash
  for _i in $(seq 10); do
    if { [ -z "$LLM_PID" ] || ! kill -0 "$LLM_PID" 2>/dev/null; } && \
       { [ -z "$STT_PID" ] || ! kill -0 "$STT_PID" 2>/dev/null; } && \
       { [ -z "$TTS_PID" ] || ! kill -0 "$TTS_PID" 2>/dev/null; }; then
      break
    fi
    sleep 1
  done
```
