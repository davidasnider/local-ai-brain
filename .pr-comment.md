## Fixes Applied (agy review round 2)

Resolved all 5 findings from the agy review:

### Critical
- **Mismatched Quotes in Python Script Invocation** (`update-deps/SKILL.md`): The closing double-quote for the `uv run python -c` block was misplaced at the end of `cfg = yaml.safe_load(f) or {}`, causing the rest of the Python code to be evaluated as Bash syntax. Moved the quote to the end of the full Python block (after the `print()` statement), so all lines are properly inside the Python string.

### Medium
- **Unsafe Process Status Checks on Empty PID Variables** (`start-backends/SKILL.md`): In `cleanup()`, guarded `kill -0` checks with `[ -z "$pid" ]` to avoid "illegal pid" errors when services fail early (empty PIDs).
- **Portability Issue with Socket Health Checks** (`start-backends/SKILL.md`): The socket health checks now use native Bash `/dev/tcp` redirection, eliminating `uv` startup overhead.

### Low
- **Insufficient LLM Startup Connection Timeout** (`start-backends/SKILL.md`): Increased startup polling retries from `seq 5` (5 seconds) to `seq 30` (30 seconds) across all three service loops (LLM, STT, TTS) to accommodate large model loading times.
- **Model Local Path Handling on Remote Updates Check** (`update-deps/SKILL.md`): Added a guard in `check_model()` that returns early for local filesystem paths (starting with `/` or `.`) that do not exist yet, preventing `git ls-remote` from being run against a non-existent local path.
