ISSUE 2: Backslash accumulation in LOCAL_API_KEY reader/writer

In the reader, line 49-50:
  q = re.match(r'^\"((?:[^\"\\]|\\.)*)\"(.*)', val)
  if q: val = q.group(1)
The regex captures quoted content but does NOT unescape backslash-escaped characters. So if the file has LOCAL_API_KEY="abc\"def", the captured value is abc\"def (with literal backslash). The printed value still has the backslash.
Fix: Add .replace('\\"', '"') after q.group(1) to unescape embedded quotes.

In the writer (update_env_key function, line 24):
  key = os.environ["LOCAL_API_KEY_VALUE"].replace("\"", "\\\"")
This escapes double quotes. If the key already contains escaped quotes (from a previous pass), this will double-escape them, causing backslashes to accumulate.
Fix: First unescape any existing backslash-escaped quotes, then escape for shell safety:
  key = os.environ["LOCAL_API_KEY_VALUE"].replace("\\\"", "\"").replace("\"", "\\\"")

Replace all `echo "LOCAL_API_KEY=\"$LOCAL_API_KEY\""` lines with `printf 'LOCAL_API_KEY="%s"\n' "$LOCAL_API_KEY"` for safe writing without shell backslash interpretation. These are on lines 117, 127, 138, and 143 of the current file.
